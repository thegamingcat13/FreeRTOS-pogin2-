Opbouw applicatie

Hieronder zie je de voor deze applicatie belangrijkste bestanden. De applicatie bevindt zich in de map Core, en wel hoofdzakelijk in de map MyApp, die weer verdeeld is in App en Ports. Daarnaast zie je in Core nog de map Src, die door STM gegenereerde bestanden bevat. 

    

  • Core/Src          [Bevat de door STM gegenereerde bestanden]
    main.c          [bevat ISR voor UART, blijf ik verder zoveel mogelijk uit...]
    stm32f4xx_it.c  [bevat ISR voor ARM_keys]
    • 

  • Core/MyApp/App    [Bevat de applicatie-code (Michiel Scager)]
    my_app.c        [versie-beheer, kleine algemene functies]
    my_app.h        [#defines, externals, etc.]
    handles.c       [alle handles worden hier aangemaakt]
    tasks.c         [alle taken (tasks, threads) worden hier aangemaakt]
    ledjes.c        [mutex, notification]
    UART_keys.c     [afhandeling van UART-input via ISR in main.c]
    ARM_keys.c      [afhandeling van ARM-toetsten via ISR in stm32f4xx_it.c]
    timer.c         [afhandeling van software-timer via handles.c]
    student.c       [speeltuin voor studenten]
    • 

  • Core/Myapp/Ports  [Bevat ARM-port-functies (Franc van der Bent)]
    buzzer.c        [buzzer-stuff]
    uart.c          [UART-stuff]
    leds.c          [ledjes]
    lcd.c           [lcd-stuff]
    keys.c          [ARM-toetsen]

    •

Getting the Gist of Markdown's Formatting Syntax


This page offers a brief overview of what it's like to use Markdown.
The [syntax page] s provides complete, detailed documentation for
every feature, but Markdown should be very easy to pick up simply by
looking at a few examples of it in action. The examples on this page
are written in a before/after style, showing example syntax and the
HTML output produced by Markdown.
It's also helpful to simply try Markdown out; the [Dingus] d is a
web application that allows you type your own Markdown-formatted text
and translate it to XHTML.
